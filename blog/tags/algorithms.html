September 3, 2024 · 6 min read
Thorsten Suckow-Homberg
info

Oft wird der Nachweis, dass das Halteproblem nicht entscheidbar ist, in der Fachliteratur (Schöning [📖Sch08, p. 119 f.], Asteroth und Baier [📖BA02, p. 106 f.], Sipser [📖Sip12, p. 216 f.]) mithilfe einer Turingmaschine und einem Widerspruchsbeweis gezeigt, in etwa:

Angenommen,

K′={w∈Σ∗∣w=⟨T⟩,T stoppt bei Eingabe w}K' = \{w \in \Sigma^* | w = \langle T \rangle, \text{$T$ stoppt bei Eingabe $w$}\}K′={w∈Σ∗∣w=⟨T⟩,T stoppt bei Eingabe w}

ist entscheidbar.

Sei T′T'T′ ist die Turingmaschine, die K′K'K′ entscheidet.
Sei w=⟨T⟩w = \langle T \ranglew=⟨T⟩ die Codierung1 einer Turingmaschine, die wie folgt arbeitet:

  • TTT simuliert das Verhalten von T′T'T′ bei der Eingabe www:
    • TTT stoppt, wenn T′T'T′ die Eingabe www verwirft (w∉K′w \notin K'w∈/K′)
    • TTT stoppt nicht, wenn T′T'T′ die Eingabe www akzeptiert (w∈K′w \in K'w∈K′)

Damit folgt: TTT stoppt bei Eingabe www ⇔\Leftrightarrow⇔ T′T'T′ verwirft www ⇔\Leftrightarrow⇔ w∉K′w \notin K'w∈/K′ ⇔\Leftrightarrow⇔ TTT stoppt nicht

Widerspruch!

Bei Vossen und Witt wird der Beweis anhand unentscheidbarer Mengen gezeigt - hierzu wird in dem Abschnitt über Berechenbarkeit die Abzählbarkeit von Turingmaschinen und analog dazu die Standardnummerierung von P\mathcal{P}P2 als theoretischer Unterbau festgelegt, um darauf aufbauend den Nachweis der Semi-Entscheidbarkeit und Unentscheidbarkeit zu führen. Im Folgenden findet sich die Beweisführung nach Vossen und Witt [📖VW16, p. 360 ff.] mit ergänzenden Kommentaren.

Die Standardnummerierung ist festgelegt als:

(N0,P,φ)(\mathbb{N}_0, \mathcal{P}, \varphi)(N0​,P,φ)
  • N0\mathbb{N}_0N0​: Die Menge aller Turingmaschinen (ein i∈N0i \in \mathbb{N}_0i∈N0​ repräsentiert die Gödelnummer einer Turingmaschine), wobei T\mathcal{T}T die Menge aller normierten Turingmaschinen ist. Die Abbildung h:N0→Th: \mathbb{N}_0 \rightarrow \mathcal{T}h:N0​→T ist definiert durch h(i)={T,falls p(τ(T))=iTω,sonsth(i) = \begin{cases} T, & \text{falls } p(\tau(T)) = i\\ T_{\omega}, & \text{sonst} \end{cases}h(i)={T,Tω​,​falls p(τ(T))=isonst​ bildet also auf eine natürliche Zahl eine Turingmaschine ab.
  • P\mathcal{P}P: Die Menge aller berechenbaren Funktionen.
  • φ:N0→P\varphi: \mathbb{N}_0 \rightarrow \mathcal{P}φ:N0​→P ist eine Abzählung aller berechenbaren Funktionen. φ\varphiφ ist festgelegt durch φ(i)=f⇔fh(i)=f\varphi(i) = f \Leftrightarrow f_{h(i)} = fφ(i)=f⇔fh(i)​=f Die berechenbare Funktion φ(i)\varphi(i)φ(i) wird also durch die iii-te Turingmaschine berechnet.

Die Standardnummerierung ist im Folgenden die Grundlage für die Formulierung von Entscheidbarkeitsproblemen bei Mengen.

Das spezielle Halteproblem​

Zunächst wird das spezielle Halteproblem (auch Selbstanwendbarkeitsproblem) betrachtet.
Vossen und Witt definieren es über folgende Menge:

K={i∣i∈Def(φi)}K = \{i \mid i \in \text{Def}(\varphi_i)\}K={i∣i∈Def(φi​)}

KKK enthält alle Nummern von Turingmaschinen, die, angewendet auf sich selbst, anhalten:

  • iii ist die Codierung einer Turingmaschine.
  • φi\varphi_iφi​ ist die berechenbare Funktion, die von der Turingmaschine p(τ(T))=ip(\tau(T)) = ip(τ(T))=i berechnet wird.
  • Def(φi)\text{Def}(\varphi_i)Def(φi​) ist der Definitionsbereich der Funktion φi\varphi_iφi​, also alle erlaubten Eingaben.
  • i∈Def(φi)i \in \text{Def}(\varphi_i)i∈Def(φi​) bedeutet: Die Codierung iii der Turingmaschine p(τ(T))=ip(\tau(T)) = ip(τ(T))=i, die φi\varphi_iφi​ berechnet, ist eine erlaubte Eingabe.

Vossen und Witt definieren KKK als die "Menge aller Turingmaschinen, die, [...] anhalten."
Das bedeutet, dass von einer entscheidbaren Menge ausgegangen wird:
Es wird angenommen, dass sowohl KKK als auch K‾\overline{K}K semi-entscheidbar sind und die charakteristische Funktion χK\chi_KχK​ von KKK berechenbar ist.

Mit Satz 10.7 formulieren Vossen und Witt den Kern des Selbstanwendbarkeitsproblems, nämlich, dass KKK zwar semi-entscheidbar, aber nicht entscheidbar ist:

Satz 10.7: [📖VW16, p. 361]
  • KKK ist rekursiv aufzählbar
  • KKK ist nicht entscheidbar

Damit wird behauptet:

  • KKK ist semi-entscheidbar, da rekursiv-aufzählbar.
    Wenn KKK semi-entscheidbar ist, ist auch die semi-charakteristische Funktion von KKK, χK′\chi'_KχK′​, berechenbar.
    Wenn KKK semi-entscheidbar und damit rekursiv aufzählbar ist, dann gilt äquivalent dazu: KKK ist der Definitionsbereich einer berechenbaren Funktion g:N0→N0g: \mathbb{N}_{0} \rightarrow \mathbb{N}_0g:N0​→N0​ (s. Folgerung 10.2 [📖VW16, p. 356]). Es reicht also aus, eine solche Funktion anzugeben, um die Semi-Entscheidbarkeit und damit auch die rekursive Aufzählbarkeit nachzuweisen.
  • Wenn KKK entscheidbar ist, dann ist auch die charakteristische Funktion χK\chi_KχK​ berechenbar: χK\chi_KχK​ gibt 111 aus, wenn die Turingmaschine TTT angewendet auf i=p(τ(T))i = p(\tau(T))i=p(τ(T)) anhält (also wenn φi(i)\varphi_i(i)φi​(i) berechnet wird); und ansonsten ist χK(i)=0\chi_K(i) = 0χK​(i)=0, wenn TTT nicht anhält3.

b) Angenommen, KKK ist entscheidbar. Dann ist χK\chi_KχK​ berechenbar.

public static int[] sort(int[] arr) {

    int n = arr.length;
    int delta = n/2;
    int min;
    int j;

    while (delta > 0) { // c1

        for (int i = delta; i < arr.length; i++) {

            // c2

            min = arr[i];
            j = i;

            while (j - delta >= 0 && min < arr[j - delta]) {

                // c3;

                arr[j] = arr[j - delta];
                j -= delta;
            }
            arr[j] = min;
        }

        delta = delta / 2;

    }
    return arr;
}

Worst-Case-Analyse​

Unter der Annahme, dass ein in umgekehrter Reihenfolge sortiertes Feld zu einer Laufzeit von O(n2)O(n^2)O(n2) bei dem Shellsort-Algorithmus führt, konnten wir mit dem in dem vorherigen Abschnitt gewählten Parametern nur eine Laufzeit von O(n log n)O(n\ log\ n)O(n log n) nachweisen.
Tatsächlich stellt der Anwendungsfall nicht den worst-case für den Algorithmus dar, da ja gerade diese Form von Sortierreihenfolge dem Algorithmus die Vorsortierung der hhh-Folgen ermöglicht:

The idea underlying Shellsort is that moving elements of A long distances at each swap in the early stages, then shorter distances later, may reduce this O(n2)O(n^2)O(n2) bound. [📖Pra72, p. 3]

Es gilt also, die Vorsortierung auszuhebeln.
Für Insertion-Sort ist die Laufzeit im worst-case O(n2)O(n^2)O(n2) (vgl. [📖OW17b, p. 87]). Da Shellsort mindestens im letzten Schritt diese Sortiermethode auf Distanzfolgen der Länge 111 anwendet, muss der Algorithmus eine Folge als Eingabe erhalten, die durch die ersten lg(n)−1lg(n) - 1lg(n)−1-Durchgänge (mit hs=2s−1,1≤s<lg(n)h_s = 2^{s - 1}, 1 \leq s < lg(n)hs​=2s−1,1≤s<lg(n)) keine Änderungen erfährt, um dann im allerletzten Schritt auf einen Schlag sortiert wird, was maximal n2\frac{n}{2}2n​ Inversionen8 bedeutet, und damit ebenso viele Vertauschungen (zuzüglich der benötigten Verschiebe-Operationen).
Hierzu kann wie im vorherigen Beispiel für n=16n=16n=16 ein Feld mit folgender Schlüsselanordnung verwendet werden: Felder mit geradem Index enthalten kleinere Schlüssel als Felder mit ungeradem Index. Hier gilt für alle Elemente aus dem Feld AAA:

∀i,j∈N[0,n−1],2∣i,2∤j:A[i]<A[j]∧A[i]<A[i+1]∧A[j]<A[j+1]\forall i, j \in \mathbb{N}_{[0, n - 1]}, 2 \mid i, 2 \nmid j: A[i] < A[j] \land A[i] < A[i + 1] \land A[j] < A[j +1]∀i,j∈N[0,n−1]​,2∣i,2∤j:A[i]<A[j]∧A[i]<A[i+1]∧A[j]<A[j+1]
