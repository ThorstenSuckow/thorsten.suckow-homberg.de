Model Matrix: Rotation - World vs. Local Origin

We introduce the mathematical foundation for rotation as linear transformations applied to model matrices in 3D computer graphics and focus on the distinction between rotating an object around an external point and rotating it around its own local origin. We derive the necessary transformations from two conceptually different but mathematically equivalent perspectives: Active transformations, which move the object in a fixed coordinate system, and passive transformations, which redefine the coordinate system around a fixed object. Additionally, we demonstrate the impact of matrix multiplication order, distinguish between world-space and local-space rotations, and conclude with performance considerations and examples from external libraries and game frameworks.

Introduction​

The effect of a rotation matrix changes significantly depending on the order of matrix multiplication, as this determines the pivot point for the operation. This article addresses two essential scenarios:

  • Rotating an object around an external point1.
  • Rotating an object around its own local origin.

We will derive the mathematical solutions for both cases. The first scenario will be explored from two distinct but equivalent perspectives: as an active transformation that directly manipulates an object's vertices, and as a passive change of coordinates that reorients the coordinate system itself.

By examining the underlying matrix compositions, we will connect this theory to practical examples in common graphics libraries.

Rotation of A around B​

The rotation of an object AAA around another object BBB can be seen as the rotation of AAA around an origin defined by BBB. For this to work, BBB must be treated as the origin.

In the following, let A,B∈R2A, B \in \mathbb{R}^2A,B∈R2 be points represented by the position vectors a⃗\vec{a}a and b⃗\vec{b}b, respectively.

Obviously, the position vectors from the world origin (0, 0) to these points are:

b⃗=B−0a⃗=A−0\begin{alignat*}{3} \vec{b} &= B - 0 \\ \vec{a} &= A - 0 \end{alignat*}ba​=B−0=A−0​

We denote the vector pointing from BBB to AAA as a⃗B\vec{a}_BaB​. This vector is calculated as:

a⃗B=A−B\vec{a}_B = A - BaB​=A−B

By subtracting BBB's position from AAA's, we obtain a direction vector a⃗B\vec{a}_BaB​ which is a direction vector from BBB - the new origin - to AAA. Applying operations to this new vector is equivalent to performing them in a coordinate system where BBB is the origin.

For this to work, three steps are necessary:

  1. Translation to Origin: The pivot point BBB is moved to the world origin. This can also be understood as a passive transformation, where the world coordinate system is shifted to align its origin with BBB.
  2. Rotation: The object AAA is then rotated relative to BBB, which is now the (temporary) origin of the world coordinate system.
  3. Translate Back: The initial translation is reversed to move the rotated object to its new position within the original coordinate system.

This is illustrated in Figure 1.

Active Transformation​

Let a⃗\vec{a}a be the position vector of the point AAA we want to rotate, and let b⃗\vec{b}b be the position vector of the pivot point BBB.

To rotate AAA around BBB, we first perform an affine transformation that translates a⃗\vec{a}a into a new coordinate system where B is the origin. This is achieved by subtracting b⃗\vec{b}b from a⃗\vec{a}a:

a⃗B=a⃗−b⃗\vec{a}_B = \vec{a} - \vec{b}aB​=a−b

The resulting vector, a⃗B\vec{a}_{B}aB​, now points from BBB to AAA. By performing this transformation, we ensure that subsequent operations, such as applying a rotation matrix R(θ)\boldsymbol{R}(\theta)R(θ), will rotate AAA relative to BBB - as if BBB were the origin.

The calculation sequence, understanding the affine transformations as matrices, is as follows:

Tb⃗ R(θ) T−b⃗ a⃗ \boldsymbol{T}_{\vec{b}}\ \boldsymbol{R}(\theta)\ \boldsymbol{T}_{-\vec{b}}\ \vec{a}Tb​ R(θ) T−b​ a

As usual, we interpret this from right to left:

translate the rotated result back←rotate the result←Translate a⃗ to the origin \text{translate the rotated result back} \leftarrow \text{rotate the result} \leftarrow \text{Translate }\vec{a}\text{ to the origin}translate the rotated result back←rotate the result←Translate a to the origin

A model matrix, M\boldsymbol{M}M, typically consists of a composition of scaling, rotation, and translation operations. We can express this as a single 4×44 \times 44×4 affine transformation matrix:

M=[Lt⃗01]\boldsymbol{M} = \begin{bmatrix} \boldsymbol{L} & \vec{t} \\ 0 & 1 \end{bmatrix}M=[L0​t1​]

Here, L\boldsymbol{L}L is a 3x3 matrix representing the combined linear transformations (scaling and rotation), and t⃗\vec{t}t is the translation vector.

To transform the object represented by this model matrix so that its pivot point BBB is at the origin, we pre-multiply its model matrix by a translation matrix T−b⃗\boldsymbol{T}_{-\vec{b}}T−b​:

T−b⃗=[I3−b⃗01]\boldsymbol{T}_{-\vec{b}} = \begin{bmatrix}\boldsymbol{I}_3 & -\vec{b}\\ 0 & 1\end{bmatrix}T−b​=[I3​0​−b1​]

By applying the rotation and corresponding back-translation, we get the final composite matrix, M

M′=[I3b⃗01][R3001][I3−b⃗01][Lt⃗01]\boldsymbol{M'} = \begin{bmatrix}\boldsymbol{I}_3 & \vec{b}\\ 0 & 1\end{bmatrix} \begin{bmatrix}\boldsymbol{R}_3 & 0\\ 0 & 1\end{bmatrix} \begin{bmatrix}\boldsymbol{I}_3 & -\vec{b}\\ 0 & 1\end{bmatrix} \begin{bmatrix}\boldsymbol{L} & \vec{t} \\0 & 1\end{bmatrix}M′=[I3​0​b1​][R3​0​01​][I3​0​−b1​][L0​t1​]

When a local-space vector a⃗\vec{a}a is multiplied by this matrix, the transformations are applied from right to left:

a′⃗=M′a⃗=[I3b⃗01][R3001][I3−b⃗01][Lt⃗01] a⃗\vec{a'} = \boldsymbol{M'} \vec{a} = \begin{bmatrix}\boldsymbol{I}_3 & \vec{b}\\ 0 & 1\end{bmatrix} \begin{bmatrix}\boldsymbol{R}_3 & 0\\ 0 & 1\end{bmatrix} \begin{bmatrix}\boldsymbol{I}_3 & -\vec{b}\\ 0 & 1\end{bmatrix} \begin{bmatrix}\boldsymbol{L} & \vec{t} \\0 & 1\end{bmatrix} \ \vec{a}a′=M′a=[I3​0​b1​][R3​0​01​][I3​0​−b1​][L0​t1​] a

This unfolds in the following sequence:

  1. Model to World: First, the local vector a⃗\vec{a}a is multiplied by the model matrix M\boldsymbol{M}M This transforms it into world space, resulting in the vector a⃗W\vec{a}_WaW​
  2. Translate to Origin: The world-space vector a⃗W\vec{a}_WaW​ is then multiplied by [I3−b⃗01]\begin{bmatrix}\boldsymbol{I}_3 & -\vec{b}\\ 0 & 1\end{bmatrix}[I3​0​−b1​]. This actively moves the vector by −b⃗-\vec{b}−b. The result is the vector's position relative to the pivot point B, a⃗B\vec{a}_BaB​.
  3. Rotate: This translated vector is then multiplied by the rotation matrix R(θ)\boldsymbol{R}(\theta)R(θ) yields a⃗R\vec{a}_RaR​, rotating it around the world origin (for a⃗B\vec{a}_BaB​, this is now BBB).
  4. Translate Back: Finally, the rotated vector is multiplied by [I3b⃗01]\begin{bmatrix}\boldsymbol{I}_3 & \vec{b}\\ 0 & 1\end{bmatrix}[I3​0​b1​]. This moves the vector back by b⃗\vec{b}b, placing it in its final rotated position in the world.

Since matrix multiplication is associative, we can pre-multiply all the transformation matrices together to create a single, final matrix M′\boldsymbol{M'}M′. Applying this single matrix M′\boldsymbol{M'}M′ to the local vector achieves the same result in one operation.

Performance Considerations

When transforming a large number nnn of vertices, pre-multiplying the transformation matrices into a single matrix is more efficient than applying each matrix to every vector step-by-step. The proof lies in comparing the total number of arithmetic operations.

Let's assume our transformation consists of the four given 4×44 \times 44×4 matrices, and we have a number of nnn homogenous 4×14 \times 14×1 vertices vi⃗\vec{v_i}vi​​. Then, the following holds in our case:

  • A dot product of two vectors v⃗i\vec{v}_ivi​, v⃗j\vec{v}_jvj​ requires 444 multiplications and 333 additions: 777 total operations.
  • A single matrix-vector multiplication requires 444 dot products, totaling 161616 multiplications and 121212 additions: 282828 total operations.
  • A matrix-matrix multiplication requires 161616 dot products, totaling 646464 multiplications and 484848 additions: 112112112 total operations.

Comparing a step-by-step transformation vs. pre-multiplying the matrices yields the following:

Step-by-Step Transformation​

For each vertex, we perform four matrix-vector multiplications. This results in a cost per vertex of 4(16+12)=64+48=1124(16+12) = 64+48 = 1124(16+12)=64+48=112 total operations. Hence, for nnn vertices, this method requires 112n112n112n operations.

Pre-Multiplying the Matrix​

This involves a one-time setup cost due to calculating M′\boldsymbol{M}'M′. To obtain M′\boldsymbol{M}'M′ we need to perform 3(4(16+12))=3(64+48)=3363 ( 4(16+12)) = 3(64+48) = 3363(4(16+12))=3(64+48)=336 total operations. Each vertex has a cost of 16+12=2816+12 = 2816+12=28 operations, hence, for nnn vertices, this method requires 336+(28n)336 + (28n)336+(28n) operations.

Solving the inequality for nnn gives us

336+(28n)<112n⇔4<n336 + (28n) < 112n \Leftrightarrow 4 < n336+(28n)<112n⇔4<n

Therefore, pre-multiplying the matrices is more computationally efficient than the step-by-step transformation for more than 444 vertices2 □\Box□

Passive Transformation​

Conceptually, the transformation can also be expressed using change-of-coordinates matrices3. In a passive transformation, the vector a⃗\vec{a}a remains fixed, and its coordinates are expressed in different coordinate systems. To incorporate translations, we represent the vector in homogeneous coordinates, (ax,ay,az,1)T(a_x, a_y, a_z, 1)^T(ax​,ay​,az​,1)T, and extend the 3×33 \times 33×3 change-of-basis matrices to 4×44 \times 44×4 affine matrices. We will assume all coordinate systems we consider have orthonormal bases.

First, we create a change-of-basis matrix to express the local vector [a⃗]A[\vec{a}]_A[a]A​ in world coordinates:

[a⃗]W=PW←A [a⃗]A[\vec{a}]_W =\underset{W \leftarrow A}{\boldsymbol{P}}\ [\vec{a}]_A[a]W​=W←AP​ [a]A​

Next, we express the world vector [a⃗]W[\vec{a}]_W[a]W​ in the coordinate system whose origin is at the pivot point, BBB:

[a⃗]B=PB←W [a⃗]W [\vec{a}]_B = \underset{B \leftarrow W}{\boldsymbol{P}}\ [\vec{a}]_W[a]B​=B←WP​ [a]W​

The rotation is applied as another change of basis. We define a new coordinate system, RRR, whose basis vectors are the columns of the active rotation matrix R\boldsymbol{R}R. This means R=PW←R\boldsymbol{R} = \underset{W \leftarrow R}{\boldsymbol{P}}R=W←RP​.

Consequently, the inverse matrix, R−1=PR←W\boldsymbol{R}^{-1} = \underset{R \leftarrow W}{\boldsymbol{P}}R−1=R←WP​, is the change-of-coordinates matrix that transforms a vector from world space WWW into our new rotated space RRR.

Using the transitivity of basis changes, we can construct the required matrix PR←B\underset{R \leftarrow B}{\boldsymbol{P}}R←BP​ as a composition of known transformations:

PR←W PW←B=PR←B\underset{R \leftarrow W}{\boldsymbol{P}}\ \underset{W \leftarrow B}{\boldsymbol{P}} = \underset{R \leftarrow B}{\boldsymbol{P}}R←WP​ W←BP​=R←BP​

We can now apply this combined matrix to find the coordinates of our vector in the final rotated system:

[a⃗]R=PR←B [a⃗]B [\vec{a}]_R = \underset{R \leftarrow B}{\boldsymbol{P}} \ [\vec{a}]_B[a]R​=R←BP​ [a]B​

By substituting the steps, we can abbreviate the entire transformation chain from the local space AAA to the final rotated space RRR:

[a⃗]R=PR←B PB←W PW←A [a⃗]A ⇔ [a⃗]R=PR←A[a⃗]A\begin{alignat*}{3} [\vec{a}]_R &= \underset{R \leftarrow B}{\boldsymbol{P}}\ \underset{B \leftarrow W}{\boldsymbol{P}}\ \underset{W \leftarrow A}{\boldsymbol{P}}\ [\vec{a}]_A\ \Leftrightarrow\ [\vec{a}]_R &= \underset{R \leftarrow A}{\boldsymbol{P}} [\vec{a}]_A \end{alignat*}[a]R​​=R←BP​ B←WP​ W←AP​ [a]A​ ⇔ [a]R​​=R←AP​[a]A​​

whereas PR←A\underset{R \leftarrow A}{\boldsymbol{P}}R←AP​ must consider all necessary translations.

The coordinates of the final vector in the world basis are obtained by applying the change-of-basis matrix using the RRR-basis expressed in world coordinates4.

[a⃗]W=PW←R[a⃗]R[\vec{a}]_W = \underset{W \leftarrow R}{\boldsymbol{P}} [\vec{a}]_R[a]W​=W←RP​[a]R​

Here, transitivity of operations applies again, so the whole transformation can be expressed by one change-of-basis matrix

[a⃗]W=PW←R PR←B PB←W PW←A [a⃗]A ⇔ [a⃗]W=P′W←A[a⃗]A\begin{alignat*}{3} [\vec{a}]_W &= \underset{W \leftarrow R}{\boldsymbol{P}}\ \underset{R \leftarrow B}{\boldsymbol{P}}\ \underset{B \leftarrow W}{\boldsymbol{P}}\ \underset{W \leftarrow A}{\boldsymbol{P}}\ [\vec{a}]_A\ \Leftrightarrow\ [\vec{a}]_W &= \underset{W \leftarrow A}{\boldsymbol{P'}} [\vec{a}]_A \end{alignat*}[a]W​​=W←RP​ R←BP​ B←WP​ W←AP​ [a]A​ ⇔ [a]W​​=W←AP′​[a]A​​

Rotating an object around its own center​

If we want to use the object's own center as the pivot point for a rotation, it is sufficient to multiply the model matrix M\boldsymbol{M}M by the rotation matrix R\boldsymbol{R}R. In this case, a separate translation to a different origin is not necessary.

The effect becomes clear when we apply the operation to a local-space vector, v⃗local\vec{v}_\text{local}vlocal​.

v⃗world=M (R v⃗local)\vec{v}_\text{world} = \boldsymbol{M}\ (\boldsymbol{R}\ \vec{v}_\text{local})vworld​=M (R vlocal​)

Interpreting this from right to left as is conventional, the vector v⃗local\vec{v}_\text{local}vlocal​ is first rotated within the object's local coordinate system. Then, the model matrix is multiplied by this newly rotated local vector, transforming it into world coordinates as v⃗world\vec{v}_\text{world}vworld​.

Since matrix multiplication is associative, we can pre-calculate a new model matrix, M′\boldsymbol{M}'M′, from the multiplication:

M′ =M R\boldsymbol{M'}\ = \boldsymbol{M}\ \boldsymbol{R}M′ =M R

This new matrix can then be applied to all vertices of the object to achieve the same result more efficiently.

rotate() in glm and Unity

The glm library provides the method GLM_FUNC_DECL mat<4, 4, T, Q> glm::rotate(mat< 4, 4, T, Q > const & m, T angle, vec< 3, T, Q > const & axis)5. This function internally calculates a rotation matrix RRR from the given angle and axis and returns the post-multiplied result MR\boldsymbol{M} \boldsymbol{R}MR. This is the standard method for applying a rotation in the local space of the input/mdoel matrix M\boldsymbol{M}M.

The Unity engine's Rotate(Vector3 eulers, Space relativeTo = Space.Self)6 method makes this choice explicit with an optional parameter. The default, Space.Self, performs a local rotation (equivalent to MR\boldsymbol{M} \boldsymbol{R}MR), while Space.World performs a world-space rotation (equivalent to R M\boldsymbol{R}\ \boldsymbol{M}R M).
