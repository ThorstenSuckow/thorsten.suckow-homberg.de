Lambda Calculus (Computer Science)

Lambda Calculus is a formal definition for expressing functional computational operations based on variable binding and substitution.

It was introduced by Alonzo Church resulting from his studies on the foundation of mathematics and computation.

Notation​

Term​

The set of λ\lambdaλ-calculus terms is defined by

t::= x∣ t1t2∣ (t) ∣ λx.tt ::= \space x | \space t_1 t_2 | \space (t) \space | \space \lambda x.tt::= x∣ t1​t2​∣ (t) ∣ λx.t

A term can be a variable (e.g., xxx), the application of one term t1t_1t1​ to another term t2t_2t2​, and an abstraction of a variable from a term ttt. Furthermore, the λ\lambdaλ-calculus allows for using parentheses for structuring terms.

Application​

(t1 t2)(t_1 \space t_2)(t1​ t2​) is an application. It is the application of the term t1t_1t1​ to an argument t2t_2t2​.

Abstraction​

λx.t\lambda x.tλx.t is an abstraction and represents a bound function with the parameter xxx and the function body ttt. xxx is bound in ttt.

λx.t\lambda x.tλx.t is also called an anonymous function. It has no name and is simply defined through the formal parameter x, prefixed with the λ\lambdaλ-symbol. A dot . separates the formal parameter from the function body ttt. If ttt contains an xxx, this xxx is bound to the formal parameter xxx.

Figure 1 This λ-abstraction represents a function that returns the sum of x and 1, whereas x is the formal parameter and bound in the term t.

Sequences of formal parameters may be collapsed:

λx.λy.λz.t⇔λxyz.t\lambda x. \lambda y . \lambda z . t \lrArr \lambda xyz.tλx.λy.λz.t⇔λxyz.t

info

The anonymous function from above can be implemented in Haskell:

Haskell
\x -> x + 1

The leading backslash ought to remind one of the Lambda-symbol: "In Haskell source code, the Greek letter lambda is replaced by a backslash character ('\') instead [...]" [Haskell Wiki]

In JavaScript, the anonymous function can be implemented like this:

JavaScript
// traditional
(function(x) {return x + 1;})

// arrow function
x => x + 1

Bound Variables​

In a Lambda term λx.t\lambda x.tλx.t we call the formal parameter λx\lambda xλx the binder, and say that the subterm ttt is in the scope of the binder: Occurrences of xxx in ttt are bound.

In an informal sense, xxx is a local variable in the function-body ttt.

The set of Bound Variables in a Lambda Calculus-term are defined by:

BV(x)=∅BV(x) = \emptyBV(x)=∅

BV(t1 t2)=BV(t1)∪BV(t2)BV(t_1 \space t_2) = BV(t_1) \cup BV(t_2)BV(t1​ t2​)=BV(t1​)∪BV(t2​)

BV(λx.t)=x∪BV(t)BV(\lambda x . t) = {x} \cup BV(t)BV(λx.t)=x∪BV(t)

bound variables in mathematics

In mathematics, bound variables are also known:

∑x=15x\displaystyle\sum_{x=1}^5 xx=1∑5​x

Free Variables​

The set of Free Variables in a Lambda Calculus-term are defined by:

FV(x)={x}FV(x) = \{x\}FV(x)={x}: x is a free variable.

FV(t1 t2)=FV(t1)∪FV(t2)FV(t_1 \space t_2) = FV(t_1) \cup FV(t_2)FV(t1​ t2​)=FV(t1​)∪FV(t2​): The free variables in the application are the union of all free variables of all sub-terms.

FV(λx.t)=FV(t)∖{x}FV(\lambda x.t) = FV(t) \setminus \{x\}FV(λx.t)=FV(t)∖{x}: The free variables in the abstraction are the free variables of the body of the abstraction, minus any variable bound by the parameters of the abstraction.

tip

A free variable is a variable that is not bound in a term ttt. The following would implement an anonymous function where the variable xxx is bound in the function body, and yyy is a free variable:

Javascript
const y = 1;

(x => x + y)(5);
Haskell
let y = 1

(\x -> x + y)(5)

β-Reduction​

Lambda calculus-terms can be brought into β\betaβ-normal-form by applying β\betaβ-reduction - that is, informal, evaluating a term by applying arguments to it.

(λx.s)t→s[x:=t](\lambda x.s) t \rarr s[x:=t](λx.s)t→s[x:=t]

Here, λx.s\lambda x.sλx.s is a term that gets applied to another term ttt, and is called the β\betaβ-redex. It gets reduced to s[x:=t]s[x:=t]s[x:=t], the reduct: All occurrences of xxx in sss are replaced with ttt.
